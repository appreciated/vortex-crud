Getting Started with Custom Repositories

1. Implement VortexCrudDataStore interface
   - Define your storage backend (filesystem, MongoDB, Redis, etc.)
   - Implement CRUD operations
   - Handle filtering and pagination

2. Create a DataStoreFactoryRegistry
   - Manage your data store instances
   - Register entity types
   - Provide data stores to the framework

3. Configure your application
   - Register your registry as a Spring bean
   - Configure VortexCrudConfiguration
   - Define routes and fields

4. Use it!
   - The framework handles the UI automatically
   - Your custom storage is now integrated

That's it! No need to create custom @Service classes or repositories.
The framework works through the VortexCrudDataStore abstraction.
