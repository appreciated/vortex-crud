Advanced Custom Repository Usage

== Lifecycle Hooks ==

Use DataStoreHooks to add custom logic:
- beforeCreate: Validate data, set defaults
- afterCreate: Send notifications, update indexes
- beforeUpdate: Check permissions, audit changes
- afterUpdate: Invalidate caches
- beforeDelete: Check dependencies
- afterDelete: Clean up related data

== Custom Data Stores ==

You can create specialized data stores for:
- Read-only access to external APIs
- Time-series data storage
- Document databases
- Key-value stores
- Graph databases
- Distributed caches

== Mixing Storage Backends ==

In one application, you can use:
- JPA for relational data (users, orders)
- Filesystem for documents
- MongoDB for unstructured data
- Redis for session storage

All through the same VortexCrudDataStore interface!

== Performance Optimization ==

- Implement caching in your data store
- Use connection pooling
- Batch operations when possible
- Lazy load related entities
- Index frequently queried fields
