# Development Platform Demo

A GitHub/GitLab-style development platform built with Vortex CRUD framework and jOOQ, demonstrating code repository management with custom fields support.

## Overview

This demo showcases a collaborative development platform with repository management, issues, pull requests, and code browsing capabilities. It demonstrates how to build complex applications with relationships, file storage, and custom field extensibility.

## Key Features

### Core Functionality
- **Repositories**: Code repository management with visibility controls
- **Organizations**: Team and organizational structure
- **Branches**: Branch management (simulated, no actual Git)
- **Files**: File tree browser with content storage
- **Issues**: Bug tracking and feature requests
- **Pull Requests**: Code review workflow
- **Milestones**: Release planning
- **Labels**: Issue and PR categorization
- **Comments**: Threaded discussions
- **Reviews**: PR approval workflow
- **Stars**: Repository favorites
- **Activity Feed**: Timeline of all actions
- **Notifications**: User notification system

### Custom Fields System ⭐

Advanced custom fields support for:
- **Repositories**: Add project-specific metadata
- **Issues**: Track custom issue properties
- **Pull Requests**: Add deployment info, breaking change flags, etc.

Same architecture as Project Management demo with JSON storage and meta table.

## Database Schema

### Core Tables

#### Repository Management
- `repository` - Code repositories with visibility, language, topics
- `organization` - Organizations owning repositories
- `organization_member` - Organization membership (M:N)
- `repository_collaborator` - Repository access control (M:N)
- `branch` - Repository branches
- `repository_file` - File tree structure with content
- `star` - Repository stars (M:N)

#### Issue Tracking
- `issue` - Issues with full workflow
- `issue_label` - Issue labels (M:N)
- `issue_assignee` - Multiple assignees (M:N)
- `milestone` - Release milestones

#### Pull Requests
- `pull_request` - PR workflow and metadata
- `pull_request_label` - PR labels (M:N)
- `pull_request_reviewer` - Code reviewers (M:N)
- `pull_request_file_change` - Files changed in PR with diffs

#### Collaboration
- `label` - Labels per repository
- `comment` - Comments on issues/PRs
- `activity` - Activity timeline
- `notification` - User notifications

#### Custom Fields
- `custom_field_definition` - Field metadata and configuration

## Getting Started

### Prerequisites
- Java 21 or higher
- Maven 3.8+

### Build and Run

1. **Generate jOOQ classes and run migrations**:
   ```bash
   cd examples/jooq-dev-platform-demo
   mvn clean generate-sources
   ```

2. **Run the application**:
   ```bash
   mvn spring-boot:run
   ```

3. **Access the application**:
   - URL: http://localhost:8082
   - Default credentials: (configured in vortex-crud security)

### Database Location
- Database file: `dev-platform-demo.db` (SQLite)
- Created automatically on first run

## Architecture Patterns

### Simulated Git Operations
This demo simulates Git functionality without actual Git:
- **Branches**: Stored in database, not Git refs
- **Files**: Content in database, not filesystem
- **Commits**: Not implemented (could add commits table)
- **Merges**: Simulated by changing PR state

### File Storage Strategy
```
repository_file table
├── Stores text content directly in database
├── Binary files could use external storage + path reference
├── Parent-child relationships create tree structure
└── Branch isolation via branch_id foreign key
```

### Activity Tracking
Every action creates an `activity` record:
```sql
INSERT INTO activity
(repository_id, user_id, action_type, entity_type, entity_id, metadata)
VALUES
(1, 1, 'opened', 'issue', 5, '{"title": "Fix bug"}');
```

## Performance Considerations

1. **File Storage**: Large files should use external storage (S3, filesystem)
2. **Search**: Add full-text search indexes for issues/PRs
3. **Activity Feed**: Paginate with indexes on timestamps
4. **JSON Queries**: SQLite JSON functions can query custom fields
5. **Tree Queries**: Recursive CTEs for file tree traversal

## Extending the Platform

### Add Commits Table
```sql
CREATE TABLE commit (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    repository_id INTEGER NOT NULL,
    branch_id INTEGER NOT NULL,
    sha VARCHAR(64) NOT NULL,
    author_id INTEGER NOT NULL,
    message TEXT NOT NULL,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP
);
```

### Add Wiki System
```sql
CREATE TABLE wiki_page (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    repository_id INTEGER NOT NULL,
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    path VARCHAR(500) NOT NULL
);
```

### Add CI/CD Pipeline
```sql
CREATE TABLE pipeline (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    repository_id INTEGER NOT NULL,
    branch VARCHAR(200) NOT NULL,
    status VARCHAR(50) NOT NULL,
    config TEXT NOT NULL
);
```
